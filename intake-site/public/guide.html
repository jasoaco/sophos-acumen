1. Installing the Extension

Prerequisites

  • Google Chrome (any recent version — Chromium-based browsers like Edge also work)
  • A Sophos Central account (any tier — the extension works with whatever you have)

Step-by-Step Installation

1

Download the Extension

Click the "📦 Download Extension" button at the top of this page. You'll get a sophos-demo-extension.zip file.

2

Unzip the File

Extract the zip file. You'll get an extension/ folder containing the manifest, scripts, icons, and built-in scenarios.

💡 Put the folder somewhere permanent — Chrome references it by path. Don't leave it in Downloads. A good spot: ~/sophos-demo-extension/ or Documents/sophos-demo/extension/
3

Open Chrome Extensions

Navigate to chrome://extensions/ in Chrome. Or: click the three dots menu (⋮) → Extensions → Manage Extensions.

4

Enable Developer Mode

Look for the Developer mode toggle in the top right corner of the extensions page. Turn it ON.

Three buttons will appear: "Load unpacked", "Pack extension", and "Update".

5

Load the Extension

Click "Load unpacked" → navigate to the extension/ folder you unzipped → click Open (or Select Folder).

You should see "Sophos Central Demo Mode" appear as a card on the extensions page.

The extension loaded in Chrome's extensions page with Developer Mode enabled.
6

Pin the Extension Icon

Click the puzzle piece icon (🧩) in Chrome's toolbar → find "Sophos Central Demo Mode" → click the pin icon (📌) to keep it visible.

You'll see a 🎯 icon in your toolbar. That's your demo mode toggle.

✅ Done! The extension is installed. You can verify by going to central.sophos.com and clicking the 🎯 icon — the popup should appear.
🚀

2. Running Your First Demo

1

Log into Sophos Central

Go to central.sophos.com and log in normally. The extension doesn't affect login.

2

Open the Extension Popup

Click the 🎯 icon in Chrome's toolbar. The popup shows:

  • Scenario dropdown — pick from built-in or custom scenarios
  • Customer Name — the name that replaces your real tenant name
  • Endpoints / Servers — scale the environment to match the prospect
3

Configure Your Demo

Select 🔴 Ransomware Attack from the dropdown. Change the customer name to your prospect's company (e.g., "Mercy Hospital"). Set their endpoint count (e.g., 4200).

4

Toggle ON & Reload

Toggle the switch ON, then click 🔄 Apply & Reload Sophos Central.

The page reloads. Now you'll see:

  • ✅ Customer name in the top-right instead of your real tenant
  • ✅ Injected alerts on the Alerts page
  • ✅ Fake cases with MITRE ATT&CK mapping on the Cases page
  • ✅ Adjusted health score and device counts
  • ✅ A floating badge at the bottom showing intercept count
Cases page showing an injected ransomware case with 24 detections, MITRE mapping, and "Contoso Healthcare" tenant name.
5

Walk Through the Demo

Navigate naturally through Sophos Central. The extension intercepts API responses on every page:

  • Dashboard — health scores, alert counts, device statistics
  • Alerts — injected alerts with realistic descriptions
  • Cases — active investigations with MITRE kill chain
  • Case Detail — timeline, impacted entities, threat graph
  • Detections — XDR detections with raw telemetry
  • Devices — industry-appropriate hostnames at scale
  • Live Discover — XDR query results
  • Audit Logs — response actions taken
6

End the Demo

Click the 🎯 icon → toggle OFF → reload the page. Your real data is back instantly.

⚠️ Safety: The extension never sends any data to Sophos Central. All write operations (isolate device, acknowledge alert, etc.) are blocked and faked — you'll see a "success" response but nothing actually happens. Your real environment is completely untouched.

4. Custom Scenarios

Importing a Scenario

You can import custom scenario JSON files into the extension:

  1. Click the 🎯 extension icon to open the popup
  2. Click 📥 Import → paste JSON, or click 📁 to load a .json file
  3. The scenario appears in the dropdown under "Custom Scenarios"
  4. Customer name and endpoint count auto-populate from the scenario's customer{} block

Exporting a Scenario

Click 📤 Export in the popup to download the currently selected scenario as a JSON file. Share it with other SEs.

Sharing via URL

After generating a scenario in the Scenario Builder, click 🔗 Share to copy a URL. When someone opens that URL, the scenario auto-imports — no file transfer needed.

Deleting Custom Scenarios

Select a custom scenario in the dropdown → the 🗑️ button appears → click it to remove. Built-in scenarios can't be deleted.

6. Advanced Features

Floating Demo Badge

When demo mode is active, a small badge appears at the bottom of the Sophos Central page showing:

  • Active scenario name
  • Number of intercepted API requests
  • A visual reminder you're in demo mode
💡 The badge is only visible to you — it's injected into the page DOM, so it doesn't appear in screenshots or screen shares unless you want it to.

Timed Events

Some scenarios include timed events — alerts that "fire" during the demo for dramatic effect. For example, in the ransomware scenario, a new alert may appear 60 seconds into the demo showing the attacker's lateral movement.

Demo Recording

The extension automatically tracks your demo session:

  • Pages visited and time spent on each
  • Number of API requests intercepted
  • Session duration

The last 50 sessions are saved in the extension's history.

Chrome Sync

Custom scenarios and preferences sync across your Chrome devices via chrome.storage.sync. Import a scenario on your laptop and it's available on your desktop too.

Scenario Library

Visit the 📚 Scenario Library to browse all built-in scenarios, see stats, and use "Make It Mine" for instant cloning with your prospect's details. You can also AI Remix any scenario for a different industry or threat actor.

7. What-If Mode — Live Alert Injection

What-If Mode lets you inject alerts live during a demo with keyboard shortcuts. Great for "let me show you what happens when…" moments.

Keyboard Shortcuts

While on any Sophos Central page with demo mode active:

  • Ctrl + Shift + 1 — Inject ransomware alert
  • Ctrl + Shift + 2 — Inject phishing alert
  • Ctrl + Shift + 3 — Inject lateral movement alert
  • Ctrl + Shift + 4 — Inject data exfiltration alert
  • Ctrl + Shift + 5 — Inject device isolation notification
💡 Pro move: During a demo, casually say "Let me show you what happens when we detect lateral movement" and hit Ctrl+Shift+3. A new alert appears in real-time. Navigate to the Alerts page and walk through it.

8. Demo Best Practices

Before the Demo

  • Test the scenario first. Toggle on, click through the key pages, make sure everything renders. Do this 30 minutes before the call.
  • Match the story to the prospect. If they're healthcare, use a healthcare scenario with HIPAA references and medical hostnames.
  • Use the AI Scenario Builder to generate a custom scenario with their exact company name and industry. It takes 30 seconds.
  • Generate a Demo Script for a suggested talk track. Even experienced SEs find the page-by-page walkthrough useful.
  • Close browser console. Demo mode logs to console — make sure F12 DevTools is closed.
  • Disable other extensions that might interfere (ad blockers, privacy tools, VPNs).

During the Demo

  • Tell a story, don't read screens. The injected data is your backdrop — narrate the attack timeline.
  • Use What-If Mode for dramatic moments. "Let me show you what happens when an attacker tries to move laterally…" Ctrl+Shift+3
  • Click into Case Detail. The MITRE ATT&CK mapping, timeline, and threat graph are the most impressive views.
  • Show scale. "You said you have 4,200 endpoints — here's what that looks like in Sophos Central."
  • Let them click around. The real UI responds to all interactions. If they ask "what's that?" — click it.

After the Demo

  • Generate a Follow-Up Email using the 📧 button in the Scenario Builder. It references specific data from the demo.
  • Generate a Battle Card if a competitor was discussed — send the key differentiators in the follow-up.
  • Toggle OFF immediately after the demo to return to your real data.

9. Safety & Troubleshooting

Safety Guarantees

  • ✅ The extension never modifies any request TO Sophos Central — only the responses coming back
  • ✅ All write operations (POST, PUT, DELETE) to action endpoints are blocked and faked
  • ✅ The extension runs entirely in your browser. No data is sent anywhere
  • ✅ Toggling off and reloading restores your real data immediately

Common Issues

Extension doesn't appear after loading

Make sure you selected the extension/ folder, not the parent folder. The folder must contain manifest.json at the root level.

Tenant name doesn't change

Make sure demo mode is toggled ON in the popup, and you clicked 🔄 Apply & Reload. Check the browser console (F12 → Console) for [Sophos Demo] 🟢 ENABLED.

Some pages show real data

A few Sophos Central pages use micro-frontends that load through non-standard mechanisms. The vast majority of pages work: Dashboard, Alerts, Cases, Detections, Devices, Threat Graphs, Live Discover, Audit Logs. If a page shows real data, navigate to one that works.

Extension conflicts with other extensions

Disable ad blockers, privacy extensions, or VPN extensions during demos. They can interfere with fetch interception.

Data looks stale

Scenario timestamps are relative (e.g., "3 minutes ago"). If you loaded the scenario hours ago, just reload the page — timestamps recalculate.

⚠️ Never demo on a customer's Sophos Central tenant. Only use this on your own SE tenant. The extension modifies what you see in your own browser, but the URL bar shows central.sophos.com — a savvy prospect might notice if anything looks inconsistent.
🔄

10. Updating the Extension

When a new version is available:

  1. Download the latest extension zip from this page
  2. Unzip and replace the old extension/ folder with the new one
  3. Go to chrome://extensions/
  4. Click the 🔄 refresh icon on the Sophos Central Demo Mode card (or click "Update" at the top)
  5. Reload any open Sophos Central tabs
💡 If you cloned the Git repository instead of downloading the zip, just git pull and refresh the extension in Chrome.
cd ~/sophos-demo && git pull origin main

Your custom scenarios are stored in Chrome's extension storage — they survive updates.
